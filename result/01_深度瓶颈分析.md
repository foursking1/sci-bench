## 三、瓶颈类型说明与改进建议

### 1. 数据未准备

**定义**: 实验产出的 metrics.json 中缺少 truth.md 要求的数据路径，且不是由于大规模训练导致。

**典型表现**:
- `value not found at task_metrics.XXX` — 实验跑过但特定指标未写入
- `expression evaluation failed` — 嵌套路径不存在

**改进措施**:
- 检查实验代码是否正确导出所有要求的指标
- 确保 truth.md 中的 target_path 与 metrics.json 中的实际键名匹配

### 2. 数值精度偏差

**定义**: 实验产出了指标值，但与 truth.md 中的目标值差距超出 tolerance 范围。

**典型表现**:
- `value=X target=Y tolerance=T` — X 和 Y 差距超过 T
- 量级错误：如 value=26.9 target=0.012

**改进措施**:
- 逐条对比 truth.md target_value 与 metrics.json 实际值
- 检查实验超参数配置是否与原文一致
- 对于百分比类指标，确认是 0-100 还是 0-1 的尺度

### 3. 文件产出缺失

**定义**: truth.md 要求的 figure/exists 规则对应的文件未生成。

**典型表现**:
- `no candidate files found for keywords: figure,XXX`
- `exists=False path=XXX`

**改进措施**:
- 确保可视化代码覆盖 truth.md 中要求的所有图表
- 检查 figure 规则中的 keywords 是否能匹配到实际文件名

### 4. LLM评估失败

**定义**: LLM 图像/文本评估报错或评分低于阈值。

**典型表现**:
- `file=XXX.png llm_score=2.0/3.0` — 评分低于阈值
- `LLM judgment failed for all candidates`

**改进措施**:
- 优化 LLM prompt，提供更清晰的 rubric
- 对于 LLM judgment failed 的情况，增加 fallback 逻辑

### 5. 执行能力缺乏

**定义**: 需要大规模训练（如 ImageNet、CIFAR-100、DDPM）的实验未能完成。

**典型表现**:
- 涉及 imagenet/cifar100/ddpm 等大模型的实验未产出结果
- 需要 GPU 长时间训练的任务中断

**改进措施**:
- 确保有足够的 GPU 资源
- 考虑使用预训练模型或缩小规模
- 增加实验 checkpoint 机制

### 6. 证据采集不足

**定义**: collect_agent 未能在论文目录中找到足够的证据文件。

**典型表现**:
- collect_report.json 中 no_evidence 数量远大于 evidence_found
- 论文目录中缺少实验结果文件

**改进措施**:
- 确保实验运行并将结果写入论文目录
- 改进文件发现和证据采集逻辑
- 对于缺少实验结果输出的论文，先补全实验运行
